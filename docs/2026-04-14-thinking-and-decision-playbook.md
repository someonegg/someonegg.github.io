# 思考与决策实践手册

> 目标：把零散概念整理成一套可执行、可复盘、可迁移的思考与行动手册。

## 1. 一页总览：从“想明白”到“做成事”

抽象为一个闭环：

1. 澄清问题（逻辑）
2. 选择模型（思维）
3. 完成论证（逻辑）
4. 组织表达（方法）
5. 推进行动（方法）
6. 复盘迭代（逻辑 + 方法）

可记为：`澄清 -> 建模 -> 论证 -> 表达 -> 执行 -> 复盘`

这 6 步里，逻辑负责“正确性”，模型负责“解释力”，方法负责“落地效率”。

## 2. 核心内容

## 2.1 方法论组件（用于沟通与执行）

### 工作沟通三原则

- 不怕说话：主动同步关键事实与风险
- 不说废话：只传递有决策价值的信息
- 不说空话：信息必须可验证、可更新

### SMART（目标定义）

- `S`pecific：目标具体，不抽象
- `M`easurable：有量化标准
- `A`ttainable：资源约束下可达成
- `R`elevant：与上层目标一致
- `T`ime-bound：有明确截止时间

### 3T（危机沟通）

- Tell it all：信息完整
- Tell it fast：信息及时
- Tell it truthfully：信息真实

### STAR（复盘与面试表达）

- Situation：背景
- Target：目标
- Action：动作
- Result：结果

### CWC（岗位匹配）

- Can you do the job?
- Will you love the job?
- Can we tolerate working with you?

### CLT（学习与表达降复杂）

- 祛除冗余
- 分割复杂
- 交替实例

## 2.2 思维模型组件（用于看清结构）

- 二阶思维：看长期连锁反应，不只看眼前收益
- 整体思维：放入场景与关系网中理解问题
- 系统思维：把输入-转换-输出视作一个可优化系统
- 第一性原理：剥离表象，从不可再分的约束出发
- IO 模型：关注变化前后状态，而非过程噪音
- SD 模型：识别供给与需求如何连接
- PO 模型：分析动力与阻力并设计推进路径
- CS 模型：区分“变化项”与“不变项”

## 2.3 逻辑组件（用于确保论证有效）

逻辑是什么：

- 缺乏基本的逻辑和思维训练，是现代教育中的常见短板。
- 逻辑不仅仅是沟通表达的工具，更是大脑处理信息与形成判断的工作方式。

1. 澄清和定义

- 问题本身究竟是什么？
- 先定义讨论对象、范围与关键术语，才能导向共识。
- 共识是有效讨论问题的基础。

2. 提炼和概化

- 通过提炼与概化，能优化信息储存和调用效率。
- 更重要的是从信息中萃取意义与价值，完成吸收内化，持续提升认知质量。

3. 推理和论证

- 有效论证方法本质上主要有两种：归纳与演绎。
- 归纳用于从多个事实抽取一般结论；演绎用于由前提推出特定结论。
- 需要警惕低质量论证方式：诉诸个例、诉诸权威、诉诸常识、诉诸群众。

4. 假设和验证

- 先基于已有材料提出假设，再围绕假设搜集数据、进行实验和观察结果。
- 根据反馈修正假设，通过反复迭代逐步接近真相。

## 3. 补充：一套可执行的“问题处理协议”

## 3.1 六步协议（可直接套用）

### 第一步：定义问题（Definition）

最小问题定义模板：

- 我们要解决的具体问题是什么？
- 影响范围是什么？谁受影响？
- 不解决的成本是什么？
- 本轮不解决什么（边界）？

产出物：`问题定义卡`（100-200 字）

### 第二步：拆解结构（Modeling）

先用 3 个模型做快速扫描：

- 第一性原理：哪些前提是不可妥协约束？
- IO：当前输入状态与目标输出状态分别是什么？
- PO：推进动力和关键阻力分别是什么？

产出物：`结构图`（约束、状态、阻力清单）

### 第三步：形成方案（Reasoning）

每个候选方案至少回答：

- 证据是什么（归纳）？
- 推导链是什么（演绎）？
- 最坏情况是什么（二阶）？
- 成立条件是什么（假设）？

产出物：`方案对比表`（方案 A/B/C + 条件 + 风险）

### 第四步：对齐沟通（Communication）

对外统一用 STAR 或 3T：

- 日常汇报：STAR（背景-目标-动作-结果）
- 异常升级：3T（完整-及时-真实）

产出物：`决策摘要`（1 页内）

### 第五步：推进执行（Execution）

执行层统一 SMART：

- 任务目标可量化
- 责任人唯一
- 时间点明确
- 验收标准提前定义

产出物：`执行看板`（Owner / Deadline / Metric）

### 第六步：复盘迭代（Review）

复盘用 CS + 二阶：

- 哪些变量改变了？
- 哪些底层规律没变？
- 当下结果在 1-3 个月后会触发什么连锁反应？

产出物：`复盘条目`（继续/停止/调整）

## 3.2 常见误区与替代动作

- 误区：目标很大但没有量化  
  替代：把目标改写成 SMART 的可验收句式

- 误区：讨论激烈但定义不一致  
  替代：先写出 100 字问题定义，再开会

- 误区：只谈方案，不谈约束  
  替代：先列不可突破约束（预算、时间、质量）

- 误区：只看短期收益  
  替代：强制补一段二阶影响分析

- 误区：结论先行，证据滞后  
  替代：按“假设-验证”顺序推进，禁止倒序论证

## 4. 高复用模板

## 4.1 决策模板（短版）

```text
问题定义：
目标（SMART）：
关键约束（第一性）：
现状 -> 目标状态（IO）：
方案 A/B：
证据与推导（归纳/演绎）：
二阶影响：
风险与对策（PO）：
最终决策：
负责人/时间/指标：
```

## 4.2 复盘模板（短版）

```text
S：发生了什么（事实）？
T：原目标是什么？
A：采取了什么动作？
R：结果如何（量化）？

改变了什么（CS-Change）？
没变的底层因素（CS-Stable）？
下一轮要验证的假设是什么？
```

## 5. 结语

“方法论”解决怎么做，“思维模型”解决怎么看，“逻辑”解决为何成立。
三者合并后，不是更多概念，而是一条可反复执行的工作流：
`定义清楚 -> 推理严谨 -> 表达简洁 -> 执行可控 -> 复盘可迁移`。
