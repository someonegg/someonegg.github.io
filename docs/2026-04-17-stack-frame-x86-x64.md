# X86 与 X64 栈帧要点

## X86

```text
X86（无 FPO）Frame Chain（按原图重画）

Stack elements:
  +----------------------------+
  | Parameter 1 for routine 1  |\
  +----------------------------+ \
  | First caller return addr.  |  >  Stack frame 1
  +----------------------------+ /
->| First caller EBP           |/
  +----------------------------+
  | Parameter 2 for routine 2  |\
  +----------------------------+ \
  | Parameter 1 for routine 2  |  |
  +----------------------------+  |
  | Return address, routine 1  |  |
  +----------------------------+  |
->| EBP value for routine 1    |  >  Stack frame 2
  +----------------------------+  |
  | Local data                 |  |
  +----------------------------+  |
  | Local data                 |  |
  +----------------------------+  |
  | Local data                 | /
  +----------------------------+/
  | Parameter 1 for routine 3  |\
  +----------------------------+ \
  | Return address, routine 2  |  >  Stack frame 3
  +----------------------------+ /
->| EBP value for routine 2    |/
  +----------------------------+\
  | Local data                 | \
  +----------------------------+ /
  | Return address, routine 3  |/
  +----------------------------+\
->| EBP value for routine 3    | \   <- Current EBP
  +----------------------------+  >  Stack frame 4
  | Local data                 | /
  +----------------------------+/
  | Local data                 |
```

在未开启栈帧优化（FPO）的情况下，可通过 `EBP` 串起帧链并完成 Frame Chain Walk。

常见相关路径：`RtlCaptureStackBackTrace`、`RtlWalkFrameChain`。

## X64

```text
X64 (two-column redraw, keep original English labels)

LEFT: Stack layout (Function A calls B)      RIGHT: Notes
----------------------------------------      ------------------------------------------
| A stack parameter stack area         |      A stack parameter stack area
|                                      |      Number of entries is equal to four or the
| A register parameter stack area      |      maximum number of parameters in any
|   R9 home                            |      function A calls - whichever is greater.
|   R8 home                            |
|   RDX home                           |      A register parameter stack area
|   RCX home                           |
| Caller return address                |
|--------------------------------------| <--- call B
| Local variables and saved            |      If used, frame pointer will generally
| nonvolatile registers                |      point here.
|                                      |
| alloca space (if used)               |
|                                      |
| B stack parameter stack area         |      B stack parameter stack area
|                                      |      Number of entries is equal to four or the
| B register parameter stack area      |      maximum number of parameters in any
|   R9 home                            |      function B calls - whichever is greater.
|   R8 home                            |
|   RDX home                           |      B register parameter stack area
|   RCX home                           |
----------------------------------------      ------------------------------------------
```

在没有动态栈分配（如 `alloca`）时，X64 通常只用 `RSP` 维护栈帧；函数入口会一次性为局部变量与子调用预留栈空间。

传给子函数的参数与返回地址位于栈底区域，调用参数槽通常至少预留 4 个位置；若函数是叶子函数（不调用子函数），可以不预留。

X64 的 Frame Chain Walk 不能仅靠寄存器链表，而是需要 `RSP` 与 PE 的 `FunctionTable`（展开信息）配合：

1. 通过函数展开信息知道当前函数的栈布局
2. 确定栈预留大小与恢复规则
3. 推导下一帧的栈位置

常见相关路径：`RtlCaptureStackBackTrace`、`RtlWalkFrameChain`、`RtlLookupFunctionTable`。

## 对比结论

1. X86（无 FPO）更依赖 `EBP` 帧链，可直接链式回溯
2. X64 更依赖 unwind 元数据（`FunctionTable`）进行可靠回溯
3. 一旦存在动态栈分配或编译优化，回溯质量高度依赖完整的展开信息
