## 核心框架：一个飞轮，四股力量

Q1 出现的四个结构性变化，不是并列关系，而是因果飞轮：

```
长程 Agent 产品化
    → 暴露"不守规矩"问题
约束工程（Harness）
    → Agent 有了纪律，才能长程运行
递归研发
    → 长程运行才能持续自我改进
Skill 生态
    → 经验可被复用，反哺 Agent 能力
    → 飞轮转入下一圈
```

筛选标准：全行业共振 + 因果咬合 + 质变被感知 + 认知不可逆。

---

## 一、长程 Agent 产品化：成年礼

**核心变化**：Agent 从"演示三步就技惊四座、走到第五步就崩"，变成了"你去睡觉，它自己在那干活"。

**关键数据**：
- Cursor Agent 单任务跑到 36 小时
- Claude Code 单日提交全球 4% 公开 GitHub 代码，年化收入约 25 亿美元
- Anthropic 内部 >90% 新代码由 AI 自主编写，52 天连发 74 次更新
- OpenClaw 60 天 GitHub 星从 9K → 247K，月活 200 万
- 国内"龙虾大战"：至少九家大厂同季推出桌面 Agent

**为什么偏偏是现在？两个条件同时到位**：
1. 模型跨过"可持续执行"及格线——局部犯错可纠正，不再全局崩溃
2. Harness 工程方法论成熟——记忆从黑盒向可编辑纯文本文件演进，执行环境有心跳/网关/浏览器接管

**六个关键判断**：

| 判断 | 要点 |
|------|------|
| 破圈靠可及性，不靠能力 | OpenClaw 靠 IM 接入 + 7×24 主动性，技术深度不如 Cowork，却拿走用户心智 |
| 五条路线同时出现 | OpenClaw（个人助理）/ Cowork（办公协作）/ Codex App（长程工程）/ Perplexity Computer（统一工作站）/ 腾讯云 ADP（企业平台） |
| 编码之后，第二个刚需场景已成立 | 13,700 个 Skill 覆盖场景远超编码；短程低认知操作不适合 Agent |
| 围墙花园挡不住 Agent | 越刚需，跨平台压力越大；硅谷打定价仗，中国打入口仗 |
| 第一层替代潮落在外包服务 | 替外包 vs 替员工，阻力相差一个数量级；6 倍于软件的服务市场是真正地基 |
| 技术能力已跑到组织接口前面 | 卡住的是：evaluation 体系、授权边界、责任归属——三样都是时间和制度的函数 |

## 二、约束工程（Harness Engineering）：让 Agent 守规矩

**背景**：从 Anthropic 2025-12-05 发布第一篇 Harness 博客，到 LangChain 2026-03-10 泛化定义，仅用 **15 周**完成行业共识。

**核心比喻**：模型是引擎，Prompt 是方向盘，Harness 是变速箱 + 仪表盘 + 刹车。

**三层壳**（每一层都被 bug 逼出来）：

| 层级 | 治什么病 | 解法 |
|------|---------|------|
| 第一层：流程管控 | 记忆金鱼、做三步就宣布完成 | 状态外化（AGENTS.md / 进度文件）+ 接力架构，Agent 之间只通过文件传信息 |
| 第二层：并发调度 | 百台 Agent 集体规避风险、挑简单的做 | 甲方乙方架构：Planner → Generator → Evaluator；Evaluator 用真实浏览器测试 |
| 第三层：验证纠错 | 自己跑自己的测试说"没 bug"但端到端跑不通（"self-deception"） | 独立 Evaluator + 沙箱隔离 + Git 事务边界 |

**效果与代价**：
- 同一模型换一套 Harness，Terminal Bench 通过率 52.8% → 66.5%，排名从三十名开外 → 前五
- 代价：裸跑 9 美元/20 分钟（主功能损坏），完整 Harness 200 美元/6 小时（功能完整可玩）—— **20 倍成本，换来"能用 vs 不能用"的生死之别**

**重要警示**：当模型变强，Harness 组件会从"帮忙"变成"碍事"。Opus 4.6 发布后，Anthropic 已拆掉 Context Reset 和 Sprint Contract，理由是"每个组件都编码了一条关于模型做不到什么的假设，假设不成立，组件就该走了"。目前完成从"加"到"拆"完整周期的只有 Anthropic 一家。

**Harness 之上**：更大的框架是 **Agentic Infra（五层）**：

> Context 层 / 工具接口层 / Harness 层 / 知识层 / 经济性层

下一阶段的战场可能在其他层：记忆质量、执行环境资源（放宽资源限制可提升 6 个百分点）、Skill 触发机制、成本控制。

## 三、递归研发：Agent 开始自己变强

**前提**：研发天然满足三条件——可验证（测试通过即通过）、可回退（Git）、可读写（纯文本代码）。

**三条路径**：

**探索型 — AlphaEvolve（Google）**
- Gemini Flash（广度）+ Gemini Pro（深度）组成进化系统
- 产出人类可读代码，非黑箱
- 已在 Google 生产环境回收全球 **0.7% 算力**（数十亿美元级别）
- 改进 TPU 关键电路速度 23%，FlashAttention 底层指令优化 32.5%
- 改进了 Strassen 1969 年的矩阵乘法算法

**优化型 — Autoresearch（Karpathy）+ M2.7（MiniMax）**
- Autoresearch：630 行 Python，三文件架构，"棘轮"规则（只保留更好结果），每小时 12 个实验，一晚 80-100 个；3 天 23K 星，3 周 35K 星
- M2.7：让模型充当"研究型 Agent"改进自己的训练流程；三层记忆（短期笔记/自我批评/历史回顾）；100+ 轮后内部评测提升 30%，SWE-Pro 达 56.22%，API 价格仅为 Claude 4.5 Sonnet 的 8%

**工程流型 — Claude/Codex 参与母公司自身研发**
- Anthropic：>90% 新代码由 AI 写，52 天 74 次更新

**结构性问题**：
- 人脑速度成为限速器——Agent 一天跑 500 轮，人拍脑袋定目标跟不上
- 复利的致命前提：评估管线必须真实，否则"在错误方向上越跑越快"（Goodhart's Law）
- 自进化 Infra 五要素：可变/不可变资产分离、评估管线、记忆与选择机制、执行环境、动态工具

## 四、Skill 生态：知识不再附着在人身上

**本质**：把"老员工脑子里的踩坑 know-how"变成 Agent 能读取和执行的结构化格式。

**Skill 的定位**：
```
Prompt（即时但不可复用）
    ↕ Skill 在两者之间
Workflow（稳定但僵硬）
```
比 Prompt 更稳（结构化+可版本控制），比 Workflow 更活（模型可灵活运用），比重训模型更轻（改 Markdown vs 重训几百亿参数）。

**规模效应**：一个资深工程师写 2 小时，全公司几千个 Agent 实例瞬间加载 —— 以前 junior 两年积累的经验，一个文件分发出去。

**生态格局**：
- **ClawHub（开放社区）**：13,700+ Skill，单个最高 18 万安装，支持 Claude/Codex/Cursor 跨平台互通
- **腾讯 SkillHub（平台审核）**：安全优先，开放性受限
- **扣子 / DeerFlow（开源可控）**：Skill = Markdown，进 Git 即有版本控制

**严重警告**：
- 341 个恶意 Skill（占 11.3%），36% 含提示词注入
- MCP 生态 60 天内爆出 30 个 CVE，82% 有路径遍历漏洞
- **56% 的情况下 Agent 根本不会主动去查自己有的 Skill**（Vercel 评测：有 AGENTS.md 索引塞进 system prompt → 通过率 100%；放在"书架"让 Agent 自己翻 → 通过率 53%，跟没给一样）

**对 SaaS 的冲击**：Skill 动摇的不是接口层，是流程层本身。当 Skill 让 Agent 跑完"用 Salesforce 管客户"的全套流程，用户就不需要 Salesforce 的界面了。

**未解问题**：经验蒸馏后人往哪退？执行层交给 Agent，决策层容不下一千个人，"递归研发正在接管 Evaluator，AlphaEvolve 正在发现人类没想到的算法"。

---

## 尾声：飞轮已转，但摩擦还大

**当前已知的结构性缺陷**：
- 341 个恶意 Skill（生态极不成熟）
- 56% 的 Skill 触发失败率
- 完整 Harness 成本动辄上千美元
- 组织治理框架空白（代码归属、审计日志、责任归属）
- 长程 Skill 会重蹈 Harness 覆辙（步骤越多，累积失败率越高）

**核心结论**：Agent 正在从"需要手把手带的工具"变成"能独立干活、守规矩、会成长、懂行的新同事"。这个速度比大多数人预期快，比大多数组织准备好的快。

> **飞轮不会等你想好了再转。**
