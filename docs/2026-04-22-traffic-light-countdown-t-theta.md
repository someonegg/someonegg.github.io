## 1. 范围与结论

- `θ`（相位零点/时间偏移）是倒计时落地必须要有的参数，但在该专利中未见完整公开算法细节；可按工程常见方法实现在线估计与校准。

---

## 2. 参数定义

- `T`：周期时长（秒），信号状态重复一轮所需时间。
- `θ`：相位零点（秒），可理解为“某一参考轮次起点”在系统时间轴上的位置。
- `φ(t)`：时刻 `t` 的当前相位，定义为：

```text
φ(t) = (t - θ) mod T
```

其中 `φ(t)` 位于 `[0, T)`。

---

## 3. 专利里关于 T 的核心算法（明确）

按 CN114463969A/B 的描述可抽象为以下流程。

### 3.1 数据筛选

在目标路口、目标时间段内，收集“停车等待目标红绿灯”的样本车辆轨迹。并可：
- 设置最小停车时长阈值，剔除非等待样本。
- 用轨迹分类模型剔除非机动车轨迹。

### 3.2 每日基准对齐（首车起步基准）

对每一天，找到该目标时段内“第一个启动”的样本车，记其启动时刻为当天基准 `t0(d)`。

实际使用时，"首车"应做异常剔除：若第一个起步样本速度异常、轨迹置信度低或停车时长过短，可顺延取下一个，或取前 N 个起步时刻的众数作为 `t0(d)`，避免单条异常轨迹导致当日所有偏移全部错位。

对同日其余样本车计算偏移：

```text
Δt_i(d) = t_start_i(d) - t0(d)
```

### 3.3 多天叠加形成偏移分布

把多天的 `Δt` 聚合统计为“偏移时长维度的数量分布”。

### 3.4 周期挖掘

在上述分布上挖掘唯一周期（专利给出可采用周期挖掘算法，如 DFT 思路），得到该路口该时段的 `T`。

### 3.5 时段聚合

同一路口在一天内不同时间段可有不同周期，最终形成“周期分布信息”（如早高峰、平峰、夜间不同 `T`）。

---

## 4. 为什么 T 要这样算

- 多天绝对时钟不一致，直接拼绝对时间会错位。
- 用“首车起步基准”做相对化后，主要保留周期结构而不是钟表时刻。
- 叠加分布后做周期检测，抗噪声能力优于单天/单轨迹直接拟合。

## 5. θ 的工程算法（必要但非该专利核心公开）

> 下述为工程化实现框架，用于解释倒计时如何“在线读秒”。

仅有 `T` 还不能倒计时，必须在线估计 `θ`（相位零点）。

### 5.1 观测建模

当系统检测到“新一轮起步波”事件，得到观测时刻 `t_obs`。

若可估计排队延迟 `δ`（例如由排队长度、车头时距估计），可先校正：

```text
t_anchor = t_obs - δ
```

`t_anchor` 更接近真实相位边界时刻（例如某绿灯起点）。

### 5.2 相位残差与递推更新

定义环形误差（wrap 到 `[-T/2, T/2]`）：

```text
e = wrap(t_anchor - θ, T)
```

在线更新：

```text
θ <- θ + K * e
```

- `K` 为增益（0~1），可固定，也可按观测质量自适应。
- 观测质量可由样本量、速度一致性、定位质量、是否拥堵溢出等构成权重。

### 5.3 倒计时计算

有了 `T` 和 `θ`，即可得当前相位：

```text
φ(t_now) = (t_now - θ) mod T
```

已知信号窗口边界（红/绿区间）后，可计算当前所处区间剩余时间与等待轮数。

**符号约定**

- 绿灯窗口在周期内的相位区间：`[g_s, g_e)`（`0 ≤ g_s < g_e ≤ T`）
- 本车前方排队车辆数（含本车）：`n`
- 每轮绿灯最大放行量：`C = floor((g_e - g_s) / h)`，`h` 为平均车头时距（秒/辆）

**等待轮数与总等待时间**

所需绿灯轮数：

```text
k = ceil(n / C)
```

根据当前相位分两种情况：

*情况一：当前在红灯（`φ ∉ [g_s, g_e)`）*

```text
d = (g_s - φ) mod T          # 到下一次绿灯起点的等待时间
W = d + (k - 1) × T          # 总等待时间
```

*情况二：当前在绿灯（`φ ∈ [g_s, g_e)`）*

```text
C_rem = floor((g_e - φ) / h)  # 本轮绿灯剩余放行量
```

- 若 `n ≤ C_rem`：本轮即可通过，`W ≈ (n - 1) × h`（按排队位置顺序估算）
- 若 `n > C_rem`：本轮放不完，令剩余未放行量 `n' = n - C_rem`，转为情况一：

```text
d' = T - (φ - g_s)            # 到下一轮绿灯起点的等待时间
W  = d' + (ceil(n' / C) - 1) × T
```

### 5.4 置信度与降级

当出现以下情况应降级：
- 残差 `|e|` 连续偏大。
- 残差方差突增（疑似手控/自适应配时突变）。
- 样本稀疏（夜间、低流量）。

降级策略：
- 不显示精确秒数，仅显示“预计等待”。
- 或直接关闭该路口读秒。

## 6. 一个可落地的最小实现框架

### 6.1 离线任务（小时级）

输入：多天轨迹样本。
输出：`T_by_intersection_by_timeslot`。

步骤：
1. 样本清洗（停车阈值、轨迹分类）。
2. 计算每日 `t0(d)` 与 `Δt`。
3. 聚合偏移分布。
4. 周期检测得到 `T` 与置信度。
5. 写入配置中心/特征库。

### 6.2 在线任务（秒级）

输入：最新轨迹流与离线 `T`。
状态：每路口维护 `θ`、残差统计、置信度。
输出：倒计时秒数与等待轮数。

步骤：
1. 检测起步波事件得到 `t_obs`。
2. 估计 `δ`（可选）并得 `t_anchor`。
3. 递推更新 `θ`。
4. 计算 `φ(t_now)` 与倒计时。
5. 置信度不足则降级。

## 7. 关键风险与对策

- 自适应信号导致 `T` 非常数。
  - 对策：改为分段 `T` 或短窗重估，低置信度时关闭读秒。
- 排队溢出使“起步时刻”偏离真实变灯时刻。
  - 对策：引入 `δ` 修正与拥堵场景单独模型。
- 右转放行/特殊渠化干扰样本。
  - 对策：车道级筛选，剔除干扰轨迹。
